# 2026-04-15 — Beta prep

Session focused on getting RegenCompliance private-beta-ready. Four priorities from the user, executed in order, auto-deployed to compliance.regenportal.com.

## What shipped

**Commit `7c7848d` — scanner: scope rules + context + URL deep-link + stacked demo**
- Compliance rules now filtered by the clinic's selected treatments. General rules (empty `applies_to`) always apply; treatment-specific rules only when `applies_to` overlaps `profile.treatments`. Fallback to all rules when treatments unset. Applied in `/api/scan`, `/api/scan-url`, `/api/scan-file`.
- `ScanFlag` type gained optional `context` (sentence containing the phrase) and `element_type` (HTML tag, URL scans only). All 4 scan-route prompts updated to return `context`; `scan-url` also returns `element_type`.
- Scanner + demo + history detail UIs render the matched phrase highlighted inside its surrounding sentence.
- URL scan results show "Found in `<tag>`" subtitle + "View on page →" link using Chrome text fragments (`#:~:text=`).
- Demo page (`app/demo/page.tsx`) was 2-column — stacked vertically.
- Login had duplicate toast + inline banner on auth failure — removed the toasts.

**Commit `39df42d` — fix view-on-page deep link**
- `lib/text-fragment.ts` helper: trims anchor to first 8 words, collapses whitespace, strips smart quotes + leading/trailing punctuation. Without this the raw `matched_text` often silently failed because cheerio-stripped text didn't byte-match the rendered DOM.
- Applied in `app/dashboard/scanner/page.tsx` and `app/dashboard/history/[id]/page.tsx`.
- Text fragments only work in Chromium (Chrome/Edge); Firefox/Safari silently ignore.

**Commit `2490812` — favicon**
- New green swirl mark across `app/favicon.ico`, `app/icon.png`, `app/apple-icon.png`. `favicon.ico` is a PNG renamed — modern browsers accept it. Convert to true ICO later if needed.
